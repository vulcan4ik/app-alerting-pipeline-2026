#  App Alerting Pipeline 2026

Проект: конвейер для автоматического формирования отчётов по метрикам приложения и реализация системы алертов для ключевых метрик сервиса сообщений и новостной ленты приложения.
Отчет формируется ежедневно за прошлые сутки  и отправляется в телеграм руководителя.    
В продакшен-режиме данные подтягиваются в реальном времени из ClickHouse (через DAG-и Airflow).

Для демонстрации предусмотрен demo-режим с чтением данных из локальных CSV.

##  Постановка задачи

**Цели:**
1. Быстро получать обзорные данные по метрикам приложения (DAU, сегменты, сообщения).  
2. Оперативно реагировать на падения ключевых метрик (CTR ленты, объём сообщений).

**Техническая задача:**  
Необходимо  **каждые 15 минут** проверять ключевые метрики: активные пользователи в ленте/мессенджере, просмотры, лайки, CTR, количество отправленных сообщений.

При существенном отклонении отправлять в телеграмм канал структурированный алерт, который содержит:
- значение метрики и виличину отклонения; 
- метод детектирования;
- график и ссылка на дашборд для исследования причин.

Ежедневный текстовый отчёт по метрикам за прошедшие сутки также формируется автоматически и отправляется в Telegram.

## 📈 Результат

- Перевел ежедневный отчёт в полностью автоматический формат: отчёт за прошедшие сутки формируется и уходит в Telegram с помощью AIRFLOW
- Ключевые метрики ленты и мессенджера контролируются каждые 15 минут автоматически, без ручного мониторинга дашбордов.
- При аномалиях руководитель получает сразу: значение метрики, величину отклонения, метод детектирования, график и ссылку на BI-дашборд


## ✅ Что было сделано

### 1. Реализация продакшн-решений в Airflow DAG

```
                ClickHouse 
                    ↓
                Airflow DAG
                    ↓
                Load (IQR / Day-over-Day)
                    ↓
                Telegram Bot API
```
**dag-app-tg-report.py** — ежедневный отчёт по метрикам приложения:
- Выборка данных из ClickHouse за прошлые сутки;
- Расчёт агрегатов (DAU, сегменты, сообщения);
- Формирование Markdown-отчёта с графиками;
- Отправка  отчёта в Telegram

**dag-alerts-report.py** — проверка алертов каждые 15 минут:
- Выборка свежих данных из ClickHouse (лента, сообщения);


 **Методы детектирования аномалий**:
  
*Метод 1: IQR (основной)*

```
  def check_anomaly_iqr(df: pd.DataFrame, metric: str, a: float = 3.5, n: int = 6:
```

- расчёт Q1 и Q3 за последние n=6 интервалов (1.5 часа)
- IQR = Q3 − Q1
- верхняя граница = Q3 + 3.5 × IQR 
- нижняя граница = Q1 − 3.5 × IQR

*Метод 2: Day-over-Day (дополнительный)*

```
def check_anomaly_day_ago(df: pd.DataFrame, metric: str, threshold: float = 0.35):
```
- сравнение с той же 15-минуткой вчера
- отклонение > 35% → аномалия
- защита от нулевых значений
- определение направления (рост / падение)

При аномалии формируется алерт с графиком, ссылкой на дашборд и отправляется в Telegram-канал.

### 2. Демо - режим генерации отчёта по приложению

- модуль `app_report_demo.py`:
  - читает входные данные;
  - считает агрегаты (DAU, активность по сегментам, сообщения и т.п.);
  - строит графики и сохраняет их в `app_charts.png`;
  - формирует текстовый отчёт `app_report.md` в Markdown.

### 3. Реализация алертов по ключевым метрикам

- Разработан модуль `alerts_demo.py` для проверки алертов:
  - **Лента (feed CTR)** — проверка CTR ленты по интервалам.
  - **Сообщения (messages)** — проверка количества и доставки сообщений.  
- Для каждого алерта:
  - считается метрика и сравнивается с порогами;
  - формируется отчёт `.md` с статусом «alert / no alert» и комментариями;
  - строится график `.png` для визуального анализа.

### 4. Скрипт запуска демо

- `run_demo.py`:
  - запускает генерацию основного отчёта по приложению;
  - выполняет проверку алертов;
  - сохраняет результаты в output/;
  - выводит список созданных файлов.

### 5. Запуск демо режима при помощи Docker
Клонируйте репозиторий через команду терминала или просто скачайте себе на локальную машину и выполните docker build  и docker run  в папке проекта

```
git clone https://github.com/vulcan4ik/app-alerting-pipeline-2026.git
cd app-alerting-pipeline-2026

docker build -f docker/Dockerfile -t app-alert-demo .
docker run --rm -v "./output:/app/output" app-alert-demo
```
